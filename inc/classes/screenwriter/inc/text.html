Features

Mockingbird is a post relationship plugin for WordPress. It allows you to manually create lists of posts, pages and custom post types for individual entries to display on your site. Packed with features, itâ€™s easy to use and is the perfect way to improve your onsite SEO and get users exploring your website. Mockingbird is WordPress 3.8 ready and has legacy support back to WordPress 3.4. At its most basic, Mockingbird can be used to display a curated list at the bottom of each post. However, its more powerful developer features are intended to allow you to create relationships between all post types set up on your website. It can be integrated and used anywhere (and for nearly any purpose) in your Wordpress theme. You could for example use it to display staff members associated with a specific department in your company. Or to curate which client testimonials you wish to display on certain project portfolio pages. Mockingbird could be used to build carousels on your e-commerce site, showing off your featured products. The possibilities are endless. Features include the ability too:
  • Automatically display page editor data on each posts.
  • Set up the amount of inputs and editors.
  • Set up input labeling.
A documented API is also provided for further theme integration, development and configuration, including the ability too:
  • Get the page editor object for the current post, page or custom post type.
  • Style the outputted list with a custom style array.
  • Create multiple Mockingbird instances with individual admin configurations.
  • Specify what post types to be displayed on.
  • Specify what individual posts to be displayed on.

Documentation - get_page_editors()

This function will return {page editors} saved data.

Usage

$page_editors = get_page_editors( $id, $key );

Parameters

$id (integer) (optional) The ID of the post from which you want to return data. Use the get_the_id() function or global $post object ID (eg $post->ID) to get the value dynamically. Default: none $key (string) (optional) A string value specifying the unique key used to save data to the Wordpress post_meta table. A unique key is required to separate data from different {page_editor} instances, preventing possible conflicts. Default: 'sc_post_type_text_editors'

Return Value

  • If $id is left blank the function returns results for the current post, page or custom post type.
  • If $id is specified, the function returns results for the specified post, page or custom post type.
  • If $key is specified, the function returns results from that {page editor} instance.

Examples

Get page editor data for current post. 
<?php $page_editors = get_page_editors(); ?>
Get page editor data for the post with the id of 5. 
<?php $page_editors = get_page_editors(5); ?>

Notes

  • Values can also be accessed manually using Wordpress's get_post_meta() function and passing 'sc_page_editors' (or a custom key value) as the specified key.

Documentation - Admin Class

{Page editor} instances can be created manually using the Page_editors_admin() class. This allows for the editors to be individually configured per post type (with different configuration options).

Usage

new Page_editors_admin( $params, $key );

Parameters

$params (array) (optional) An array of specified configuration values for the {page editor} instance. This provides a framework for how the instance should behave. The configuration options are:
  • post_types - an array of post types to be displayed on. Defaults to array("post").
  • title - a string to display as the metabox title. Defaults to "Page Content".
  • priority - priority level of the metabox (default, core, high, low). Defaults to "default".
  • context - position of the metabox (normal, side, advanced). Defaults to "side".
  • show_on - an array of post ids to be displayed on. Defaults to array().
  • editors - an array of configuration arrays outling the editors. Defaults to array("post").
  • config - an array of configuration options for theTinyMCE plugin. Defaults to array().
Default: array() $key (string) (required) A string value specifying the unique key used to save data to the Wordpress post_meta table. A unique key is required to separate data from different {page_editor} instances, preventing possible conflicts. Default: none

Examples

Create an additional {page editor} instance, with different configuration settings. 
<?php
	$args = array(
		'config' => array(
		'toolbar' => 'bold italic underline'
		),
		'editors' => array(
			array(
				'key' => 'first_editor',
				'label' => 'First Editor'
			),
			array(
				'key' => 'second_editor',
				'label' => 'Second Editor'
			)
		)
		);

	new Page_editors_admin( $args, "page_content" );
?>

Notes

  • If a unique key is not supplied the {page_editor_admin} class will override the initial plugin instance with any new configuration set in the $params array.
  • As this function creates a new instance, any saved {page_editor_admin} settings will not be applied to the new instance. Any intended deviation from the default settings has to be configured manually using the $params array.
  • Due to some confilicts with the TinyMCE plugin it is not possible to mainatin the default text editor on pages where the {page editors} are in use.
